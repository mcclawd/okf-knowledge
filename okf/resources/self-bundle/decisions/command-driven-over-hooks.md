# Decision: command-driven over hooks

**Context.** To keep a bundle in sync, one option was to fire updates automatically (Claude Code
hooks). But a skill is *passive* — it cannot reliably run "at the end of a task" — and hooks run
commands, not model judgment, so they cannot author concept prose.

**Decision.** Make okf-knowledge an explicit **[`/okf` command](/systems/okf-command.md)** (like
`/graphify`): the user runs `/okf update` when they want a sync.

**Why.** Explicit beats implicit here — it is reliable (no dependence on the model remembering)
and simple (no hook wiring). For real automation, pair it with a CI
[`validate`](/systems/validator.md) gate; deterministic, code-derived concepts can additionally
be regenerated by a script.
